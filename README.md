# Prompt Engineering Guide

A simple and practical guide to understanding prompt engineering — the skill of communicating effectively with AI.

This repository is based on classroom learning, refined into clear notes, examples, and techniques that actually make a difference.

---

## What is a Prompt?

A prompt is the input or instruction given to an AI model to generate a response.

Every interaction follows a basic flow:

Input → AI → Output

The quality of the output depends on how clearly the input is written.

Example:

Bad prompt:
"Tell me about Python"

Better prompt:
"Explain Python for a beginner in 100 words with examples"

---

## Types of Prompts

- **Zero-shot prompting**  
  Asking without giving examples.

- **Few-shot prompting**  
  Providing examples to guide the response.

- **Chain-of-thought prompting**  
  Encouraging step-by-step reasoning.

- **Role-based prompting**  
  Assigning a role (e.g., "Act as a data scientist").

---

## Components of a Good Prompt

A strong prompt usually includes:

- Clear instruction  
- Context  
- Constraints (length, format)  
- Tone or style  

Simple formula:

Role + Task + Context + Format + Constraints

---

## Techniques

- Ask for step-by-step explanations  
- Use clear separators like quotes or symbols  
- Provide examples when needed  
- Break complex problems into smaller parts  
- Refine and iterate based on output
